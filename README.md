Простое веб-приложение в docker
===

## Описание

Использует 2 контейнера:

1) стандартный контейнер `nginx`
2) контейнер, основанный на `php:7.1-fpm` из одного из прошлых занятий
(используется он, поскольку в нем есть настроенный composer)


## Запуск

```bash
docker-compose build
docker-compose up
```

Приложение пробрасывает порт nginx 80 на порт 8000 хост-машины

Проверку я выполнял с помощью curl на хост-машине, обращаясь к приложению на порту 8000.

```bash
curl -X GET -i 127.0.0.1:8000 # 400 (не POST-запрос)
curl -X POST -i 127.0.0.1:8000 # 400 (в запросе нет поля string)
curl -X POST --data "string=()" -i 127.0.0.1:8000 # 200
curl -X POST --data "string=(()()()()))((((()()()))(()()()(((()))))))" -i 127.0.0.1:8000 # 400
curl -X POST --data "string=()()()(())" -i 127.0.0.1:8000 # 200
curl -X POST --data "string=((())()()(()))" -i 127.0.0.1:8000 # 200
```

Возвращает код 400 если запрос не POST, если в запросе нет поля string, если в поле string находится неверная
последовательность открывающихся и закрывающихся скобок. В остальных случая возвращается код 200
