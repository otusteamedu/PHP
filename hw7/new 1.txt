1. Модульные тесты:
	
	1.1 Если длинна card_number не равна 16, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.2 Если card_number содержит не только цифры, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.3 Если card_holder содержит цифры, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.4 Если card_holder содержит какие либо символы кроме пробела и дефиса, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.5 Если card_holder содержит какие либо символы не латиницей, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.6 Если card_holder содержит более одного пробела, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.7 Если card_expiration не соответствует формату мм/гг, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.8 Если card_expiration месяц меньше 1 или больше 12, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.9 Если card_expiration месяц и год меньше текущих, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.10 Если длинна строки cvv не равна трем, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.11 Если cvv содержит не цифры, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.12 Если длинна строки order_number больше 16, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.13 Если order_number содержит пробелы, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.14 Если sum не содержит запятой, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.15 Если sum присутствуют буквы, то
	тестируемый метод возвращает 400 с сообщением об ошибке;
	
	1.16 Если sum присутствуют пробелы или иные символы кроме запятой, то
	тестируемый метод возвращает 400 с сообщением об ошибке;

	3.1 Если cvv неверен, то после получения ответа от бэка на фронте
	выводится сообщение об ошибке «Не удалось списать данные с
	карты, проверьте реквизиты».
	
	3.2 Если card_number неверен, то после получения ответа от бэка на фронте
	выводится сообщение об ошибке «Не удалось списать данные с
	карты, проверьте реквизиты».

	3.3 Ожидаем что после успешного списания денег в заказе с номером N 
	в базе данных заказ имеет поле is_paid равным true 

	3.4 Ожидаем что после успешного списания денег 
	пользователь был редиректнут на страницу благодарности
	
	3.5 Ожидаем что после успешного списания денег
